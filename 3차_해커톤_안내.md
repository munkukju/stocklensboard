# 3차 해커톤: 실무 풀스택 웹 서비스 구축 및 클라우드 배포 챌린지

본 해커톤은 8시간이라는 제한된 시간 내에 기획부터 배포까지의 전체 서비스 사이클을 직접 완수하는 것을 목표로 합니다. 단순히 로컬 환경에서의 코드 구현에 그치지 않고, 클라우드 인프라 설정과 보안 프로토콜을 적용한 실제 서비스 가능 수준의 결과물을 도출합니다.

참가자들은 React 및 Next.js(SSR/SSG/ISR 및 Client Components)를 활용한 사용자 중심의 UI 구현과, Express(`server.js`) 또는 Next.js Route Handlers를 이용한 체계적인 백엔드 설계, 그리고 MariaDB 중심의 데이터 설계 및 CRUD 처리 역량을 금융 분야의 주제를 통해 종합적으로 검증하게 됩니다.

## 핵심 수행 목표

- **프론트엔드**: React/Next.js (App Router)와 Tailwind CSS를 활용한 효율적인 인터페이스 구축
- **백엔드**: Node.js (Express) `server.js` 구조 또는 Next.js Route Handlers 중 선택하여 API 라우팅 및 비즈니스 로직 구현
- **데이터베이스**: MariaDB를 활용한 CRUD SQL 쿼리 구사 및 데이터 관리
- **인프라 및 Ops**: GCP VM 인스턴스 구축 및 Cloudflare를 활용한 SSL 보안 배포
- **협업 및 기록**: GitHub를 통한 소스 코드 관리와 기술적 문제 해결 과정을 담은 개발 보고서 작성

## 1. 해커톤 개요 및 주제

참가자는 **금융(Finance) 분야** 내에서 자유롭게 주제를 선정하여 8시간 이내에 개발 및 클라우드 배포를 완료해야 합니다. 모든 주제는 **사용자 인증(JWT)**과 **MariaDB 연동을 통한 CRUD 구현**을 필수 요건으로 합니다.

### 금융 분야 주제 예시

- 개인 자산 관리(PFM) 대시보드
- 가상화폐 또는 주식 모의 투자 플랫폼
- 금융 상품(예금/적금) 비교 및 추천 서비스
- 소비 패턴 분석 기반의 가계부 서비스

### 필수 요건

- **핵심 미션**: JWT를 활용한 보안 로그인 체계 구축 및 MariaDB를 이용한 금융 데이터의 생성, 조회, 수정, 삭제(CRUD) 구현
- **백엔드 구성**: 기존의 Node.js + Express (`server.js`) 환경을 유지하거나, Next.js의 Route Handlers (`app/api/...`)를 사용하여 서버리스 API를 구성할 수 있음
- **기술 포인트**: Next.js의 SSR/SSG/ISR 적절한 활용, 사용자별 데이터 접근 권한 검증, SQL을 활용한 동적 데이터 처리

### 가점 사항

- 상태 관리 도구(Zustand, Jotai 등)의 체계적인 활용
- React Query (TanStack Query)를 이용한 서버 상태 관리 및 데이터 페칭 최적화 적용
